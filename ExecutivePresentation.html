02 / 10

The opportunity

CSX runs on green screens.
People shouldn't have to.

Engineers and conductors book leaves through TECS. Local chairmen coordinate weekly vacation bids by seniority. Crew callers and dispatchers live inside the MC02-family screens. Managers approve, escalate, audit. Today every one of them learns the screen shortcuts, the field positions, the auto-tab rules, and the quirks of the host. That's a tax we pay every shift.

Today

One person, one screen, one keyboard

Each request hand-typed into TECS. Errors creep in. Bulk asks become escalations. Audit lives in someone's head.

With Lucid Relay

One sentence. Anywhere. Audited.

"Cancel my April 30 vacation." Spoken, typed, emailed, or said in Teams. The agent reads the live screen, plans, confirms, commits, and files the audit.

Tomorrow

Delegation, automation, intelligence

Managers delegate. Bulk processes run unattended. Insights surface from data the host always had, but no-one could read at scale.

We don't replace the mainframe. We replace the distance between our people and the mainframe — with conversation, voice, audit, and guard rails. — Lucid Relay product principle

The scenarios

Same engine. Three very different days.

EC

The engineer

Day-to-day · mobile, hands full, in a yard or on the line
1. Opens Teams on a phone, types "book April 30 daily vacation, my last comp day."
2. Lucid checks balance, holiday calendar, blackout — confirms intent in plain English.
3. Engineer replies "yes." Lucid signs into MCVD, types the D, captures the host response.
4. Confirmation lands back in the same Teams thread. Audit row written.
No mainframe screen learning curve. No mistyped empid. Total wall-clock: 30 seconds.
MN

The manager

Delegating · approving on behalf of a team of forty
1. Asks Lucid in the desktop app: "who on my crew has unused PL going into July?"
2. Lucid reads MCPL for every direct report, returns a sorted list with carry-over dates.
3. Manager picks three names, says "draft a personal leave request for these on July 5 — wait for each person's approval."
4. Each engineer gets a notification on their channel. As they accept, Lucid commits each one.
A coordination pattern that today takes hours of phone calls — done with one sentence and three taps.
DP

The dispatcher / crew caller

High-volume · cross-screen · time-pressured
1. "Lucid, mark off all conductors on roster 1402 for tomorrow's mark-up."
2. Lucid reads MCRS for the roster, drives MCSV for each entry, surfaces conflicts before committing.
3. One confirmation gate ("commit 23 of 23?"). Lucid runs the bulk operation, screen by screen, with retries.
4. Result: a CSV summary, a per-record audit trail, and a Teams card the local chairman can countersign.
23 transactions, fully auditable, in a single voice command. No keypunch. No lost rows.

07 / 10

Beyond a single transaction

Bulk. Multi-screen. Coordinated.
With guard rails on every step.

A daily vacation request is the smallest unit. The same engine handles hundreds of records across many screens — but never silently. Every batch surfaces a plan, a confirmation, and a per-record outcome. Failures don't get swept under a rug; they become a queue of follow-ups for a human, with the rest already done.

Pattern A · Bulk same-screen

"Process this CSV against MCVD."

100 daily-vacation rows uploaded by Crew Ops. Lucid validates each (employee exists, day is selectable, balance available), shows a plan ("96 will commit, 4 will fail with reasons"), and on approval runs the batch with retries. Each row gets its own audit entry.

Pattern B · Cross-screen workflow

"Schedule weekly vacation by seniority for roster 1402."

Reads MCRS for seniority, checks MC22 for limits, walks MCVR for the schedule, falls back to MCSV when entries don't fit, and writes adjustments to MCPL where needed — all in one orchestrated session.

Pattern C · Approval chain

"Manager-initiated PL on behalf of three engineers."

One manager intent fans out to three Teams notifications. Lucid waits for each person's confirmation, commits per-employee, and reports a consolidated outcome to the manager.

Pattern D · Read-and-summarise

"What's the next 30 days look like for my crew?"

Read-only walk across MCVD/MCPL/MCRS for 40 employees, returned as a natural-language brief with anomalies flagged. Zero mainframe writes, full audit trail of what was read.

08 / 10

Built for an audited, regulated environment

Security is a first-class capability — not a feature.

Lucid Relay does not weaken any control already in place. The mainframe remains the system of record. Identity remains owned by Okta and RACF. What we add is traceability — every conversation, every decision, every keystroke logged and queryable.

Identity is end-to-end

Channel verifies the user (Okta, Teams, signed email), agent verifies the role (RACF), screen verifies the empid. No anonymous writes.

Confirmation is mandatory

Nothing reaches the host until the user (or a delegated approver) explicitly confirms in plain language. Read-only requests don't need it.

Plans are visible

Every commit is preceded by a one-line plan in the same conversation. The user sees what will happen before it happens.

The host's words are sacred

We never paraphrase mainframe responses. Successes and errors come back verbatim — UPDATE COMPLETED, NO UPDATES WERE REQUESTED — alongside our friendly summary.

Audit is automatic

Every turn captures: who, channel, intent, plan, confirmation, host response, timestamp. Queryable per employee, per day, per screen.

Secrets stay local

RACF passwords and PINs are never sent to the language model. Voice transcription runs on-device. The host session is the user's session.

Authorisation, in plain terms

  • You only see what your RACF role allows you to see.
  • You only commit what your RACF role allows you to commit.
  • Managers delegate — they don't impersonate.
  • Bulk operations require an elevated scope, signed off explicitly.

"Can I see what I've done?"

  • Yes — every user has a personal board of today's actions across all channels.
  • Filter by date, by screen, by channel, by outcome.
  • Export an audit pack for any time window.
  • Replay any conversation, including the host's verbatim responses.
