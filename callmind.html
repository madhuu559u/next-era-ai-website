Processing Pipeline

How raw transcripts become structured intelligence â€” step by step.

1

Transcript Ingestion

Raw call transcripts are ingested from the call recording platform. Metadata (call date, agent ID, LOB, duration) is extracted and normalized.

2

LLM Analysis

Each transcript is passed through GPT-4o with a structured extraction prompt. The model extracts: caller intent, sentiment score, key topics, questions asked, resolution status, and automation eligibility.

3

Entity & Pattern Extraction

Named entities (member IDs, claim numbers, provider names, medications) are identified. Process patterns and friction points are tagged for trend analysis.

4

Vector Embedding

Each processed transcript is embedded using Azure OpenAI Embeddings and stored in ChromaDB for semantic search. Enables natural language querying over the entire corpus.

5

Structured Storage

Extracted metadata and analytics are stored in SQL Server for fast aggregation. Raw transcripts and AI outputs go to Azure Blob for archival.

6

Dashboard & Q&A Sync

Analytics Dashboard pulls from SQL Server for real-time visualization. Agentic Q&A uses ChromaDB for semantic retrieval with SQL for statistical queries.

âš¡ Processing Speed

The pipeline processes transcripts in parallel using multi-threaded workers. A batch of 10,000 transcripts can be fully processed and indexed within hours, not days.
