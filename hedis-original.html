Solution Architecture & Implementation

Modular, loosely-coupled design for maximum flexibility

High-Level System Architecture
Plugin Module Architecture

üîå True Plugin Architecture

New measures can be added by simply creating a JSON configuration file and a processor class. No changes to the core engine are required. The factory pattern automatically routes processing based on measure type.

Processing Workflow

End-to-end chart processing pipeline

Processing Sequence Flow

Detailed Processing Steps

1

Chart Ingestion

Charts are uploaded to CHART.MASTER table with Blob URL and measure type identifier

2

Queue Management

Processor queues new charts to AI2_EXTRACTION table with READY status

3

Worker Claim

Multi-threaded workers claim records, update status to INPROGRESS with worker ID

4

PDF Download & OCR Verification

Download PDF from Azure Blob, verify extractable text using Tesseract (open source OCR)

5

Measure Detection & Processing

Factory pattern routes to correct measure processor (COL, BPD, WCC, etc.)

6

AI Extraction

Azure OpenAI (GPT-4o-mini) extracts keywords, answers questions, identifies coordinates

7

JSON Generation & Upload

Create highlights.json, summary.json, qa.json and upload to Azure Blob container

8

Database & Salesforce Update

Update AI2_EXTRACTION with URLs, mark COMPLETE, update Salesforce Chart object
