üíº Business Challenge

Clinical reviewers manually process thousands of medical charts annually for HEDIS quality measure compliance. The previous vendor solution had unpredictable per-page OCR costs, over-highlighting issues, poor user experience, and no code ownership. Reviewers spent excessive time navigating large PDFs to find relevant clinical information.

Technical Approach

1

Chart Ingestion

Charts uploaded to CHART.MASTER table with Azure Blob URL and measure type (CBP, BPD, COL, WCC, etc.)

2

Multi-threaded Processing

Worker threads claim records from queue, download PDFs, verify text extraction using open-source Tesseract OCR

3

Measure-Specific Extraction

Factory pattern routes to correct processor (COL, BPD, WCC). Each measure has JSON config for keywords and Q&A

4

LLM Processing

Azure OpenAI GPT-4o extracts clinical data, answers measure questions, identifies exact text coordinates

5

JSON Output Generation

Creates highlights.json, summary.json, qa.json with coordinate data uploaded to Azure Blob container

6

Salesforce Integration

Updates Chart object with AI URLs. LWC sidebar displays highlights, Q&A with one-click page navigation

Technology Stack

Azure OpenAI GPT-4o LangChain LangGraph Multi-Agent Python Flask Admin Console Azure Blob Storage SQL Server Salesforce LWC Tesseract OCR

üîå Plugin Architecture

Each measure is defined by a JSON config file specifying: measure code, relevant keywords, extraction questions, page filtering rules, and output format. Adding a new measure involves creating a config file and mapping it in the factory ‚Äî no core code changes needed. This enables rapid expansion to new document types.
