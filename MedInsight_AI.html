System Architecture

Modular, loosely-coupled design built for maximum flexibility, reliability, and extensibility across healthcare data extraction use cases.

High-Level System Architecture
Plugin Module Architecture (Factory Pattern)

üîå True Plugin Architecture

The factory pattern automatically routes processing based on measure type. New measures require only a JSON configuration file and a processor class ‚Äî no modifications to the core engine, orchestrator, or data pipeline. This enables rapid expansion to new quality measures and entirely new extraction domains (Risk Adjustment, Stars, Quality Programs) with minimal development effort.

Processing Pipeline

End-to-end automated pipeline from chart ingestion through AI extraction to clinical reviewer delivery.

Processing Sequence Flow

Pipeline Steps

1

Chart Ingestion

Charts uploaded to CHART.MASTER with Blob URLs and measure type identifiers. Supports batch ingestion and real-time arrivals.

2

Queue Management & Worker Claim

Multi-threaded workers atomically claim records (READY ‚Üí INPROGRESS) with worker ID tracking. Configurable concurrency (default: 12 threads).

3

PDF Download & OCR Verification

Source PDF downloaded from Azure Blob. Tesseract verifies extractable text quality. Pages with insufficient text quality are flagged.

4

Measure Detection & Factory Routing

Factory pattern inspects the measure type and routes to the appropriate processor plugin (COL, BPD, WCC, etc.). Each plugin has tailored prompts and extraction logic.

5

AI Extraction via Azure OpenAI

GPT-4o-mini extracts keywords with coordinate positions, answers compliance questions with clinical reasoning, and identifies relevant source pages.

6

JSON Generation & Upload

Structured outputs (highlights.json, summary.json, qa.json) generated and uploaded to Azure Blob. URLs stored in database for Salesforce access.

7

Salesforce Delivery

Chart object updated with AI URLs. Salesforce LWC sidebar renders highlights, Q&A, and timeline with one-click page navigation for reviewers.
